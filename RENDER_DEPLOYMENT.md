# Render Deployment Guide

## Prerequisites

Before deploying to Render, ensure:

1. ✅ All migrations are created and committed
2. ✅ `requirements.txt` contains all dependencies
3. ✅ `render.yaml` is configured
4. ✅ Environment variables are set

## Deployment Steps

### 1. Create Render Account & Connect Repository

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Sign up/Login
3. Click "New +" → "Blueprint"
4. Connect your GitHub repository
5. Select the repository containing this project

### 2. Deploy Using render.yaml (Recommended)

Render will automatically detect `render.yaml` in the `backend` directory and configure everything.

**Important**: Make sure `render.yaml` is in the `backend` directory, not the root.

### 3. Manual Configuration (If not using render.yaml)

#### Create PostgreSQL Database

1. In Render Dashboard, click "New +" → "PostgreSQL"
2. Name: `mams-db`
3. Database: `mams`
4. User: `mams_user`
5. Plan: Free (or your preferred plan)
6. Click "Create Database"
7. Note the connection details

#### Create Web Service

1. Click "New +" → "Web Service"
2. Connect your repository
3. Configure:
   - **Name**: `mams-backend`
   - **Root Directory**: `backend`
   - **Environment**: `Python 3`
   - **Build Command**: `chmod +x build.sh && ./build.sh`
   - **Start Command**: `gunicorn --config gunicorn_config.py mams.wsgi:application`

#### Set Environment Variables

In the Web Service settings, add these environment variables:

```
SECRET_KEY=<generate-a-secure-secret-key>
DEBUG=False
USE_SQLITE=False
DB_NAME=<from-postgres-database>
DB_USER=<from-postgres-database>
DB_PASSWORD=<from-postgres-database>
DB_HOST=<from-postgres-database>
DB_PORT=5432
ALLOWED_HOSTS=mams-backend.onrender.com
CORS_ALLOWED_ORIGINS=https://your-frontend-domain.vercel.app
```

**Note**: Replace `your-frontend-domain.vercel.app` with your actual Vercel frontend URL.

### 4. Update CORS After Deployment

After your backend is deployed:

1. Note your Render backend URL (e.g., `https://mams-backend.onrender.com`)
2. Update `CORS_ALLOWED_ORIGINS` in Render environment variables with your Vercel frontend URL
3. Update `ALLOWED_HOSTS` if needed

### 5. Run Initial Setup

After first deployment, you may want to run:

```bash
# Via Render Shell or SSH
python manage.py createsuperuser
python manage.py shell < setup_initial_data.py
```

Or use Render's "Shell" feature in the dashboard.

## Common Issues & Solutions

### Issue: Build Fails

**Solution**: 
- Check that `requirements.txt` exists and has all dependencies
- Ensure `build.sh` is executable
- Check build logs for specific errors

### Issue: Database Connection Error

**Solution**:
- Verify all database environment variables are set
- Check that PostgreSQL database is created
- Ensure `USE_SQLITE=False` is set

### Issue: Static Files Not Found

**Solution**:
- `collectstatic` runs in `build.sh`
- Check that `STATIC_ROOT` is set correctly
- Verify static files are collected in build logs

### Issue: CORS Errors

**Solution**:
- Update `CORS_ALLOWED_ORIGINS` with your frontend URL
- Ensure frontend URL includes `https://`
- Check that `CORS_ALLOW_CREDENTIALS=True` is set

### Issue: 500 Internal Server Error

**Solution**:
- Check Render logs for detailed error messages
- Verify all environment variables are set
- Ensure migrations have run (`python manage.py migrate`)

## Environment Variables Reference

| Variable | Description | Example |
|----------|-------------|---------|
| `SECRET_KEY` | Django secret key | Auto-generated by Render |
| `DEBUG` | Debug mode | `False` |
| `USE_SQLITE` | Use SQLite (local) | `False` |
| `DB_NAME` | PostgreSQL database name | `mams` |
| `DB_USER` | PostgreSQL user | `mams_user` |
| `DB_PASSWORD` | PostgreSQL password | Auto from database |
| `DB_HOST` | PostgreSQL host | Auto from database |
| `DB_PORT` | PostgreSQL port | `5432` |
| `ALLOWED_HOSTS` | Allowed hostnames | `mams-backend.onrender.com` |
| `CORS_ALLOWED_ORIGINS` | Frontend URLs | `https://your-app.vercel.app` |

## Testing Deployment

1. **Health Check**:
   ```bash
   curl https://mams-backend.onrender.com/api/assets/bases/
   ```

2. **Authentication Test**:
   ```bash
   curl -X POST https://mams-backend.onrender.com/api/token/ \
     -H "Content-Type: application/json" \
     -d '{"username":"youruser","password":"yourpass"}'
   ```

3. **Frontend Integration**:
   - Update `VITE_API_URL` in frontend `.env` to your Render backend URL
   - Test login and API calls from frontend

## Security Notes

- ✅ Never commit `.env` files
- ✅ Use strong `SECRET_KEY` in production
- ✅ Set `DEBUG=False` in production
- ✅ Restrict `ALLOWED_HOSTS` to your domain
- ✅ Use HTTPS (Render provides this automatically)
- ✅ Keep dependencies updated

## Support

If you encounter issues:

2. Review Django error logs in Render dashboard
3. Test locally with same environment variables
